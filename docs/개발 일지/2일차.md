오늘은 모델의 거리와 방향에 맞춰 점들을 이동시키고 원근투영을 통해   
모델상 좌표를 화면산 좌표로 변환하는 `VertexShader`를 구현하였다.  

## VertexShader.h

### TFVertex
```c++
struct TFVertex {
    Vec4 position;
    Vec2 texCoord;
    Vec4 normal;
    float invW{};
};
```
행렬곱을 통해 변환된 자표를 저장하기 위한 구조체이다.  
점의 위치 좌표인 `position`, 텍스쳐 상의 좌표인 `texCoord`, 법선 벡터 `normal`이 멤버로 있다.
추가로 모델 좌표에 변환행렬을 곱해 얻은 동차좌표의 역수인 `invW`가 있다.

### Shader
```c++
class Shader {
public:
    Mat44 model;
    Mat44 view;
    Mat44 projection;
    Mat44 mvp;

    void MVP() {
        mvp = projection * view  * model;
    }

    [[nodiscard]] TFVertex VertexShader(const Vertex& in, int width, int height);

};
```
멤버 변수로는 `model`, `view`, `projection`, `mvp`가 있다. 이 세 변수들의 역할을 다음과 같다.  
- `model` : 모델의 위치, 회전 등의 데이터를 저장한 행렬
- `view` : 카메라의 위치와 각도 등의 데이터를 저장한 행렬
- `projection` : 원근 투영의 정도와 범위를 저장한 행렬  

멤버 함수 `MVP()`는 `model`, `view`, `projection` 세 멤버 변수를 곱하여 클립 좌표를 구해 멤버 변수 `mvp`에 저장한다.


## VertexShader
`Shader`의 멤버 함수 `VertexShader()`는 정점과 화면의 너비, 높이를 입력받아 모델을 스크린 좌표로 변환하여 `TFVertex` 형식으로 반환한다.  

```c++
const Vec4 clip = mvp * in.position;

float invW = 1;
if (clip.w != 0) invW =  1 / clip.w;
const Vec4 ndcP = clip * invW;
```
우선 mvp와 모델의 점 벡터를 곱하여 클립 좌표로 변환한다.
그 뒤, 클립 좌표를 동차 좌표값인 `clip`의 w값으로 나누어 원근 투영시킨다.

```c++
out.position.x = (ndcP.x + 1.f) * 0.5f * static_cast<float>(width);
out.position.y = (1.f - ndcP.y) * 0.5f * static_cast<float>(height);
out.position.z = ndcP.z;
out.position.w = clip.w;
```
이후 위와 같이 투영 완료된 ndc 점들을 화면에 크기에 맞추어 스크린 좌표로 변환한다.  
ndcP 상으로는 y가 커질수록 더 위쪽에 위치하는 데카르트 좌표계로 점이 표현되었지만  
나는 배열처럼 위쪽 끝이 0, 내려갈수록 y가 커지는 C/C++ 배열 스타일로 캔버스를 만들 계획인지라  
여기서 y좌표 계산에 `(1.f - ndcP.y)` 처럼 순서를 바꾸어 위아래를 뒤집어 주었다.
