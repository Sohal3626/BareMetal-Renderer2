# 3일차 - Barycentric & Canvas

오늘은 삼각형을 색칠하기 위하여 점의 위치를 체크하기 위해 데카르트 좌표를 삼각형 내의 바리센트릭 좌표로 변환하는 함수 `barycentric()`  
과 렌더링된 이미지를 저장하기 위한 `Canvas` 구조체를 추가했다. 이제 최소한의 ppm 생성 테스트가 가능하다.

## barycentric()
barycentric 좌표는 삼각형 내부에서의 점의 상대적인 위치를 표현하기 위한 좌표계이다.  
삼각형 (P1, P2, P3)에 대한 점 p의 바리센트릭(이렇게 읽는거 맞겠지?)좌표는 아래와 같은 식으로 정의된다.

$p = aP_1 + bP_2 + cP_3 일 때, (a, b, c)$  
여기서 $a + b + c = 1$이어야 하므로 $a = (1 - b - c)$로 치환할 수 있다.

이를 기존의 식에 대입하면  

$p = (1-b-c)P_1 + bP_2 + cP_3$  

이걸 정리하면,  

$p - P_1 = b(P_2 - P1) + c(P_3 - P_1)$  
따라서, $ b(P_2 - P_1) + c(P_3 - P_1) + (P_1 - p) = 0$  

점의 데카르트 좌표는 (x, y)꼴의 형태이므로 barycentric 좌표 또한  
x에 대한 좌표와 y에 대한 좌표로 나누어 계산된다. 따라서 이 식을 x성분, y성분으로 나누면

$(x) :  b(P_{2x} - P_{1x}) + c(P_{3x} - P_{1x}) - (P_{1x} - p) = 0$
$(y) :  b(P_{2y} - P_{1y}) + c(P_{3y} - P_{1y}) - (P_{1y} - p) = 0$

와 같은 연립방정식을 푸는 문제로 생각할 수 있다.
```c++
static Vec3 barycentric(const Vec2 p, const std::span<const Vec2, 3> pts) {
    const Vec3 vx = {pts[1].x - pts[0].x, pts[2].x - pts[0].x, pts[0].x - p.x};
    const Vec3 vy = {pts[1].y - pts[0].y, pts[2].y - pts[0].y, pts[0].y - p.y};
    const Vec3 u = vx.cross(vy);
    if (std::abs(u.z) < 1e-2) return {-1, 1, 1};

    float u_val = u.x / u.z;
    float v_val = u.y / u.z;

    return {1.f - (u_val + v_val), u_val, v_val};
}
``` 
연립방정식을 푸는 방법은 여러가지 구현법이 있지만 이번엔 외적을 이용한 방법을 사용하였다.  
우선 변수 `vx`, `vy`를 통해 두 연립방정식의 계수 벡터를 정의해 주었다.  
삼각형의 꼭짓점의 좌표를 이용한 $(P_{2x} - P_{1x})$ 과 같은 형태의 식을 상수 A, B, C와 같은 형태로 단순화하면  

$A + bB + cC = 0$

와 같은 형태로 표현할 수 있다. 이어서 이 식은 두 벡터의 내적을 표현한 식으로 해석할 수 있는데

$(1, b, c) \cdot (A, B, C) = 0$

과 같은 식이다. 따라서 미지수 벡터 $(1, b, c)$를 찾는다는 것은 벡터 $(A, B, C)$에 수직인 벡터를 찾는것과 같다.
 
우리의 목적은 계수벡터 `vx`, `vy`로 이루어진 연립방정식을 푸는 것이므로 벡터 `vx`, `vy`와 동시에 수직인 벡터가 이 방정식의 해가 된다.  
두 벡터와 모두 수직인 벡터를 만드는 방법에는 두 벡터를 외적하는 방법이 있다.
따라서 그 두 벡터를 외적하여 구한 벡터는 이 방정식의 해 벡터가 된다.  

이때 외적의 결과는 실제 해에 상수를 스칼라곱 한 형태로 나타날 수 있다.  
이를 보정하는데에 있어서 해 벡터의 3번째 원소가 1이라는 점을 주목하자.  
외적의 결과 $u$가 해 벡터의 스칼라배 형태로 나타난다면 아래와 같이 표현할 수 있다.  

$u = (k, kb, kc) _{(단, k는 상수)}$  

따라서 벡터 u에 k로 스칼라 나눗셈 연산을 해주면 원래 형태의 해 벡터를 얻을 수 있다.

## Canvas.h

함수 하나만 던져놓고 "오늘 일 다 끝났다!" 하면서 놀기 찝찝해서 간단한거 하나 더 만들었다.  

```c++
struct Canvas {
    int width = 0, height = 0;
    std::vector<Vec3> pixels;

    Canvas(int w, int h);
    void setPixel(int x, int y, Vec3 color);
    void save_ppm(const std::string &filename);
};
```
그림이 그려질 판 역할을 하면서, ppm형식으로 변환되어 파일로 저장하는 역할을 하는 구조체이다.  
멤버 변수로는 캔버스 크기를 저장할 `width`, `height`가 있고, 그려지는 픽셀을 저장하는 벡터 `pixels`가 있다.
캐시 히트율을 높여서 소소하게 성능을 높여주기 위해 일차원 배열로 평탄화하여 관리한다.  

멤버 함수로는 생성자 `Canvas()`, 픽셀에 점을 찍는 함수 `setPixel()`, 캔버스를 ppm 포맷으로 저장하는 `save_ppm`이 있다.  

### `save_ppm()`   
캔버스에서 딱히 볼 만한 내용이 없어서 이것만 보겠다.  
```c++
void save_ppm(const std::string &filename) const {
        std::ofstream file(filename);
        if (!file.is_open()) {
            std::cerr << "Unable to open file! " << filename << std::endl;
            return;
        }
        file << "P3\n" << width << " " << height << "\n255\n";

        for (const auto& p : pixels) {
            file << static_cast<int>(255.99 * p.x) << " "
                << static_cast<int>(255.99 * p.y) << " "
                << static_cast<int>(255.99 * p.z) << "\n";
        }
    }
```
사실 이 부분도 보다시피 별 내용 없다. 단순히 파일을 열어서 읽고, 안되면 에러 메세지를 출력,  
아니면 ppm 파일을 생성하는 식이다.  
ppm은 이미지의 픽셀 하나하나를 기록한 텍스트 파일인데 확장자를 `.ppm`이라고 저장하고 맨 윗줄에 헤더를 쓴 뒤 아래부터는 점의 색상을 적어주면 된다.  
ppm의 헤더는 아래처럼 생겼다.  
```text
P3
(너비) (높이)
255
```
그리고 이 아래 줄부터 아래처럼 이미지의 (0, 0)좌표부터 (높이, 너비) 좌표까지 차례대로 RGB값이 나온다.
```text
155 155 155
111 111 111
222 222 222
123 123 123
.
.
.
213 111 132
```
이것들을 그냥 fstream으로 써주었다.  
근데 여기서 중요한건 RGB값은 0부터 255 사이의 값을 가져야 한다는 것이다. 
하지만 이 프로그램에서는 색상 데이터를 [0, 1]범위의 실수값으로 처리한다.  
따라서 255.99를 곱하여 포맷을 맞춰주었다.

이제는 캔버스에 실제로 그림을 그릴 시간이다.

## Rasterizer()
점 3개(삼각형의 세 꼭짓점)을 입력받아 캔버스에 삼각형으로 색칠하는 함수이다.  
`barycentric()`을 이용하면 어떤 점이 삼각형의 안쪽에 있는지, 바깥쪽에 있는지를 구분할 수 있기 때문에  
이를 이용하여 안쪽에 있다면 색칠, 바깥쪽에 있으면 스킵하는 형태이다.
```c++
static void FillTriangle(Canvas& canvas, const std::span<const TFVertex, 3> pts) {
    // 1. 삼각형이 있을 법한 범위를 대략적으로 추정

    // 2. pts에서 position 정보를 추출하여 Vec2가 3개 들어간 컨테이너로 합치기 (barycentric()을 이용하기 위함)

    // 3. 1에서 구한 범위를 순회하며 픽셀마다 barycentric()을 호출하여 삼각형의 안인지 바깥인지 구별
    
    // 3-1. 안쪽일 경우 Fragment Shader를 호출하여 색상 계산 후 점 찍기

}
```
대충 구조를 생각하면 위와 같다.  

```c++
float minX = pts[0].position.x, maxX = pts[0].position.x;
float minY = pts[0].position.y, maxY = pts[0].position.y;

for (size_t i = 1; i < pts.size(); i++) {
    minX = std::min(minX, pts[i].position.x);
    minY = std::min(minY, pts[i].position.y);
    maxX = std::max(maxX, pts[i].position.x);
    maxY = std::max(maxY, pts[i].position.y);
}

const int xStart = std::max(0, static_cast<int>(floor(minX)));
const int xEnd = std::min(canvas.width - 1, static_cast<int>(ceil(maxX)));
const int yStart = std::max(0, static_cast<int>(floor(minY)));
const int yEnd = std::min(canvas.height - 1, static_cast<int>(ceil(maxY)));
```
1 단계 부분이다.  
삼각형의 세 점의 좌표를 입력받아 삼각형이 들어갈 수 있는 사각형 범위를 구한다.  
x와 y의 최소, 최댓값을 찾아 삼각형의 외접 사각형을 구하는 식이다.  
이렇게 찾아낸 최소, 최댓값들을 다시 정수형으로 캐스팅하여 캔버스의 포맷과 맞추어 준다.

```c++
const Vec2 tri[3] = {
    Vec2 {pts[0].position.x, pts[0].position.y},
    Vec2 {pts[1].position.x, pts[1].position.y},
    Vec2 {pts[2].position.x, pts[2].position.y}
};
``` 
2 단계 부분이다.  
단순히 `TFVertex`의 `position`변수에서 값을 가져와 Vec2 배열로 만들어 주었다.

```c++
for (int i = yStart; i <= yEnd; i++) {
    for (int j = xStart; j <= xEnd; j++) {
        const Vec3 bary = barycentric(Vec2{static_cast<float>(j), static_cast<float>(i)}, tri);

        if (bary.x < 0 || bary.y < 0 || bary.z < 0) continue;

        const Vec3 color = {bary.x, bary.y, bary.z}; // 임시값 그라데이션
        // color = 프레그먼트 셰이더 호출
        canvas.setPixel(j, i, color);
    }
}
```
마지막 3단계이다.  
for 루프로 1단계에서 구한 범위를 픽셀 하나하나 순회한다.  
`barycentric()`을 점마다 호출하여 삼각형 내부의 상대적인 점의 위치를 구한다.  
여기서 하나의 변수라도 음수 값이 나온다는 것은 삼각형의 외부에 있다는 의미이므로 `continue`한다.  
이후 Fragment Shader를 호출하여 그 픽셀의 색깔이나 텍스쳐를 계산하면 되지만 아직은 Fragment Shader를 구현하지 않아  
임시로 rgb 가중치를 바리센트릭 좌표로 만들어 그라데이션처럼 나오게 만들었다.  
이후 계산된 색깔의 픽셀을 캔버스에 찍으면 된다.
