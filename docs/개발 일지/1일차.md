## Mesh.h
`.obj`파일은 가장 대표적이고 기본적인 3D모델 포맷중 하나이다.  
3D모델이라고 무슨 대단한 게 있을 것 같지만 사실을 텍스트 파일이다. 
대충 구조는 다음과 같다.
```markdown
v로 시작하는 줄        <-- 모델 상 한 정점의 좌표  
예) v 0.424663 -0.649937 -0.567418

vt으로 시작하는 줄     <-- 색깔을 뽑아올 텍스쳐파일에서의 좌표
예) vt  0.500 0.997 0.000

vn으로 시작하는 줄     <-- 밝기를 계산하기 위한 법선 벡터
예) vn  -0.001 0.661 0.751

키워드 usemtl         <-- 아래 나올 폴리곤들이 사용할 머터리얼의 이름
예) usemtl Hair

f로 시작하는 줄        <-- 모델 상 한 폴리곤의 정보 (차례대로 v/vt/vn 순서이며 값이 아닌 .obj 파일 상의 인덱스 번호)
예) f 101/79/101 112/90/112 102/80/102
```
Mesh.h는 이 `.obj`파일의 텍스트 데이터를 읽어와 프로그램이 계산하기 편하도록 데이터를 정리하여 저장한다.

### Vertex
```c++
struct alignas(16) Vertex {
    Vec4 position;
    Vec4 normal;
    Vec2 texCoord;
    float padding[2];

    bool operator==(const Vertex& v) const {
        return position == v.position && texCoord == v.texCoord && normal == v.normal;
    }
};
```
정점의 정보를 저장하기 위한 구조체이다.  
오브젝트의 위지 정보인 `positsion` 광원 정보인 `normal`, 텍스쳐 좌표인 `texCoord`를 저장한다.  
여기서 `positsion`과 `normal`에 길이가 3이 아닌 4짜리 벡터를 사용한 것을 볼 수 있는데  
이는 추후 좌표 변환과 투영 단게에서 행렬 연산을 더 편하게 하기 위함이다.  

살면서 처음 써보는 기능을 써봤는데 `alignas()`다.  
CPU는 데이터를 읽어 올 떄 메모리를 4 또는 8 바이트 단위로 잘라서 가져온다고 한다.  
`alignas(16)` 라는 키워드를 구조체에 달아서 생성하면 구조체의 시작 주소가 16의 배수로 고정된다고 한다.  
따라서 구조체가 다른 캐시 라인에 나뉘어 저장될 확률이 줄어들고  
캐시 히트율이 높아져 소소하게 성능이 빨라지는 효과가 나타난다고 한다.  

`padding` 변수도 이를 위한것인데, `alinas(16)`을 통해 데이터를 16바이트 단위로 정렬하였으므로  
이 데이터의 크기는 16바이트의 배수가 되어야 한다.  
그러나 float 4개짜리 구조체 2개와 float 2개짜리 변수 1개, 총 40byte를 사용하게 된다.  
여기서 40보다 큰 16의 가장 작은 배수는 48, 즉 `float` 2개 크기만큼의 배열을 추가로 선언하여 구조체의  
마지막 주소를 16의 배수로 만들어주었다.  
사실 이런 마지막 주소 정렬은 `alinas()`를 사용할 때 자동으로 이루어진다고 하지만
나는 처음 해당 키워드를 처음 사용하기도 하고, 학습 목적의 프로젝트이기 때문에 명시적으로 `padding`을 정의해보았다.

### FaceVertex
```c++
struct alignas(16) FaceVertex {
    int position;
    int texCoord;
    int normal;
    FaceVertex (const std::initializer_list<int> list){
        auto it = list.begin();
        while (it != list.end()) {
            if (it != list.end()) position = *it++;
            if (it != list.end()) texCoord = *it++;
            if (it != list.end()) normal = *it++;
        }
    }
    auto operator<=>(const FaceVertex&) const = default;
};
```
모델을 처리하는 과정에서 임시로 생성되는 점을 저장하기 위한 구조체이다.  
`position`은 정점 위치를, `texCoord`는 텍스쳐상의 좌표를, `normal`은 법선 벡터 정보를 저장한다.  
여기에서 `Vertex`와 다른점은 `Vertex`는 실제 데이터를 저장하고, `FaceVertex`는 `.obj`파일 상에서  
면 정보에 나오는 인덱스들을 저장한다는 점이다.

### Mesh
```c++
class alignas(16) Mesh {
public:
    std::vector<Vertex> vertices;
    std::vector<uint32_t> indices;

    struct MeshGroup {
        uint32_t indexStart;
        uint32_t indexCount;
        uint32_t materialIndex;
    };
    std::vector<MeshGroup> subMeshes;

    bool load_obj(const std::string& filename);
private:
    void set_indexStart();
};
```
모델 데이터를 직접 추출하고 `Vertex` 데이터로 조립하여 저장하는 클래스다.  
렌더러 쪽에서 그림을 그릴 때 사용할 점의 집합 `vertices`와 그 점의 인덱스를 저장하는 `indices`변수를 넣어뒀다.
이런식으로 인덱스를 저장해두면 중복 데이터를 재사용 하기 편해진다.  
추가로 `MeshGroup`이라는 구조체를 만들어 넣어주었는데 이는 정점의 인덱스를 머터리얼의 종류별로 분류하기 위함이다.
`load_obj()`와 `set_indexStart()`는 이어지는 구현부에서 설명하겠다.

### set_indexStart
```c++
void Mesh::set_indexStart(){
    if (!subMeshes.empty()) {
        subMeshes.back().indexCount = static_cast<uint32_t>(indices.size()) - subMeshes.back().indexStart;
    }
}
```
이 부분은 그리 길지 않아서 통채로 가져왔다. 머터리얼의 종류가 바뀔 때 마다 `indices`를 기준으로   
`subMeshes`의 이전 인덱스의 `indexCount`를 기록하는 역할이다.
특별한 이유가 있기 보다는 그냥 `load_obj()`   
내부에서 2번정도 반복되는 이유로 분리해 주었다.

### load_obj()
나는 전통적(?)으로 스트림 파싱을 겁나게 못한다.  
하지만 이전 프로젝트의 모델 파서, 머터리얼 파서에 이어 3번째로 이 짓거리를 하다 보니 나름 익숙해지는 것 같아 뿌듯하다.  
```c++

in.open(filename, std::ifstream::in);
if (in.fail()) {
```

일단 시작하자마자 파일 체크부 터 해 주었다. 이후 사용할 변수들을 선언해주었다.  
우리가 폴리곤을 해석하기 위해서는 정점의 정보를 저장해야 하는데,  
이를 `vertices`에 저장하기 전 임시로 들고 있을 변수가 필요한데 그 이유는 우리가 폴리곤 데이터를 추출하는  
`.obj`파일의 f로 시작되는 줄에는 실제 데이터값이 아니라 데이터의 인덱스가 들어있기 때문이다.  
따라서 아래와 같이 변수를 선언해주었다.
- `positions` 정점의 위치 좌표를 저장
- `uvs` 정점의 텍스쳐 좌표를 저장
- `normals` 정점의 법선 벡터를 저장  

그리고 나중에 이 친구들에 데이터들이 마구 `push_back()`으로 들어오면 재할당 비용이 너무 커질 것을 염려하여  
공간을 1000개 정도 미리 할당해 두었는데 알고 보니 이는 데이터 크기 대비 굉장히 소심한... 크기라고 한다.  
미리 선형탐색으로 파일의 크기를 잰 후 그에 맞춰 데이터를 미리 할당하는 방법도 있다고 한다.  
나중에 확장으로 고려해 볼만 한 것 같다.

`FaceVertex`를 키로 가지는 맵 `checkUnique`가 있는것을 확인 할 수 있는데 이는 중복으로 나오는 정점을   
재활용하기 위함이다. 처음에는 삼각형 이상의 다각형을 처리할 때 중복 정점을 처리하기 위한 의도였지만 굳이  
삼각형 이상의 다각형이 아니더라도 정점을 공유하는 상황은 충분히 흔하기 때문에 생각보다 큰 절약효과가 있었다.  
중복처리 방식의 선택지는 2가지였다.  
1. `Vertex`를 키로 사용하는 `unordered_map` 사용하기.  
2. `FaceVertex`를 키로 사용하는 `map` 사용하기.  

이 중에서 후자를 선택한 이유는 `unordered_map`을 사용하기 위해서는 별도의 해싱 함수를 작성하여야 하기 떄문이다.  
Geometry.h의 구조체에는 별도의 해싱 함수가 없기 때문에 직접 작성해 주어야 `Vertex`의 해시함수를 작성할 수 있다.  
이런 상황에서 해싱함수보다는 렌더링 파이프라인 학습에 집중해 일단 함수를 완성하고 결과를 보는게 우선이라고 생각해  
2번의 방법을 채택했다.
````c++
if (!line.compare(0, 7, "usemtl ")) {
    // 이전 머터리얼의 개수를 MeshGroup.indexCount에 기록하고 머터리얼을 매핑 
}else if (!line.compare(0, 2, "v ")) {
    // positions 에 값 저장
}else if (!line.compare(0, 3, "vt ")) {
    // uvs 에 값 저장
}else if (!line.compare(0, 3, "vn ")) {
    // normals에 값 저장
}else if (!line.compare(0, 2, "f ")) {
    // 0. 이 면이 소속될 MeshGroup이 없다면 빈 그룹을 만들기
    // 1. 점 3개 합쳐서 FaceVertex 만들기
    // 2. FaceVert 점 3개를 면 하나로 만들어 Vertex 생성하기 (다각형 -> 삼각형 분할)
    // 2-1. 만약 점 3개 중 `checkUnique`에 이미 사용된 점이 있다면 그 점을 재활용하기
    // 3. 처음 사용하는 점은 `checkUnique`에 기록하기
}
````
이후의 `if`문들은 매일 먹던 맛이니 넘어가고 f로 시작하는 줄을 발견한 경우만 자세히 보겠다.

일단 먼저 모델에서 어떤 머터리얼도 사용하지 않을 경우 머터리얼의 개수가 0으로 되는 경우가 발견돼  
현재 `subMeshes`에 `MeshGroup`이 하나도 없는 경우 기본 그룹을 하나 생성해 준다.
```c++
// 1
string segment;
vector<FaceVertex> fvs;
fvs.reserve(3);
while (iss >> segment) {
    Vertex vertex;
    ranges::replace(segment, '/', ' ');
    istringstream ss(segment);
    FaceVertex fv{-1, -1, -1};

    ss >> fv.position;
    if (ss >> fv.texCoord) {}
    if (ss >> fv.normal) {}

    if (fv.position > 0) fv.position--;
    if (fv.texCoord > 0) fv.texCoord--;
    if (fv.normal > 0) fv.normal--;
    fvs.push_back(fv);
}
```
루프를 돌며 그 줄의 점들로 `FaceVertex`를 만들어 벡터에 집어넣는다.
```c++
// 2, 2-1, 3
for (size_t i = 1; i < fvs.size() - 1; ++i) {
    FaceVertex tri[3] = { fvs[0], fvs[i], fvs[i+1] };
    for (const FaceVertex& fv : tri) {
        if (!checkUnique.contains(fv)) {
            Vertex vertex;
            Vec3 fvPos = positions[fv.position];
            vertex.position = {fvPos.x, fvPos.y, fvPos.z, 1.};
            if (fv.texCoord > -1) vertex.texCoord = uvs[fv.texCoord];
            if (fv.normal > -1) {
                Vec3 fvNormal = normals[fv.normal];
                vertex.normal = {fvNormal.x, fvNormal.y, fvNormal.z, 0.};
            }

            uint32_t idx = static_cast<uint32_t>(vertices.size());
            vertices.push_back(vertex);
            indices.push_back(idx);
            checkUnique[fv] = idx;
        }else{
            indices.push_back(checkUnique[fv]);
        }
    }
}
```
이후, 삼각형 이상의 다각형인 폴리곤이 있다면 삼각형으로 쪼개서 `Vertex`로 변환해야 하는데,  
벡터 `fvs`의 `FacVertex`들을 점을 하나 고정시켜두고 다른 점을 순서대로 2개씩 조합해 삼각형 모양으로 자르는 식이다.
이 때, 맵 `checkUnique`를 확인하여 이미 사용한 적이 있는 점이라면 그 점을 재활용, 아니라면 새 점을 맵에 넣는다.
